💡 雲端同步常見錯誤排查 (Failed to Fetch / 404)：

  • HTTP 404 錯誤（找不到部署網頁）：代表您的 Google Apps Script 網頁應用程式網址已被刪除、未設定，或是複製錯誤。請至您的 Apps Script 視窗中，點選右上角「部署」->「管理部署」，複製最新產生的「網頁應用程式 URL」（網址尾端必須是 /exec），重新在此處貼上並儲存。
  • 設定「任何人」可存取（最關鍵）：請至您 Google Apps Script 的編輯器點擊右上角「部署」->「管理部署」，將「誰可以存取（Who has access）」設定為 「任何人 (Anyone)」。如果設定為「僅限我自己」或「擁有 Google 帳號的人」，會被 Google 強制導向至驗證頁，瀏覽器會因安全政策阻擋連線並噴出 Failed to fetch。
  • 第一次執行需要授權：請先手動用瀏覽器直接開啟您的 Apps Script 網址，並同意 Google 帳號權限（進階 -> 仍要前往），否則 API 無法工作。
  • 伺服器冷啟動延遲：若長達數小時未存取，Google 伺服器冷啟動需耗時 10~15 秒，本更新已將連線逾時時間提高至 25 秒，可耐心等待重試。
  • 關閉瀏覽器跨網站防護：若您使用 Brave 或 Safari 瀏覽器，請關閉其防護（Shields/跨網站追蹤阻擋）以免正常的 iframe 背景連線被瀏覽器阻斷。
